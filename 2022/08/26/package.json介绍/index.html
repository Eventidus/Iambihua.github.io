package.json 介绍

  1. package.json 作用
    1. 包管理配置文件，管理项目中所依赖的包的配置
  2. package-lock 作用
    1. 本质是锁定(lock)包的版本
  3. 常见属性
    1. script
      1. 定义脚本命令
      2. 每次运行 npm run xxx 命令时，会创建一个 shell 脚本，这个脚本会把当前项目的 node-modules/.bin 子目录写入到系统的 Path 环境变量中
    2. bin
      1. 将一个包的可执行文件加载到全局环境中
      2. 如果要在全局环境中直接使用，前提是包要安装在全局环境下
    3. type
      1. 用来说明包使用的模块化规范
      2. 如果采用 ESM 模块规范，则 type 的值为 module
      3. 如果采用 CommonJS 模块规范，则 type 的值为 commonjs
      4. 如果不想修改 type，可以修改文件后缀名
        1. ESModel 模块规范后缀名为.mjs
        2. CommonJs 模块规范后缀名为.cjs
    4. main&module&browser
      1. 都是用来指定入口文件
      2. main：定义了包的入口文件,支持 node 和 browser 环境
      3. module 定义了包在 ESM 规范下的入口文件，支持 node 和 browdeer 环境
      4. browser 定义了 browser 环境下的入口文件
      5. 这三个入口文件的加载顺序，一般是 ESM 规范的文件先行
        1. browser+mjs>module>browser+cjs>main
    5. exports
      1. 定义 npm 包的所有导出
      2. 条件引用
        1. 引用方式以及模块化文件类型会影响入口文件的引入
        2. require 引入.cjs 文件
        3. import 引入.mjs 文件
    6. dependencies&devDependencies
      1. dependencies:项目所需要的依赖
      2. devDependencies:项目所需要的模块，用来提高开发效率
    7. engines
      1. 指定项目能够运行的 node 以及 npm 版本
  4. npm 扁平化
    1. npm 扁平化的原因是 npm2 过多重复依赖导致的磁盘浪费问题以及 window 系统下路径过长问题
    2. 缺点
      1. 幽灵依赖问题（引入没有在 package.json 中声明的依赖）
      2. 存在一个包的多个版本下，只能实现一个版本的扁平化
