Vuex

  1. 作用：集中管理项目中组件公用的数据

  2. 使用场景：项目过大，组件多，并且数据量多的情况下可以使用 vuex

  3. vuex 的状态是响应式的

  4. mutations 不能直接更改状态

  5. Vuex 使用步骤： 1. 创建 store 目录，在这个目录下创建 index.js 文件 2. 导入 Vue 及 Vuex 库 3. 安装 Vuex 库，即 Vue.use(Vuex) 4. 创建 vuex 实例，通过 new Vuex.Store({})方法 5. 向外导出 Vuex 实例 6. 在入口文件中导入 vuex 实例 7. 注册 vuex 实例

  6. vuex 的几个核心概念

    1. state
      1. 类似于 vue 的 data 属性，存储数据源
    2. mutations
      1. mutation 是同步事务
      2. 类似于 vue 实例的 methods 属性，修改数据源状态，但不能处理异步方法
      3. 不能直接修改 state 的值，可在子组件中通过调用 this.$store.commit(‘xxx’)来更改 store 中 state 的值
      4. mutation 相当于注册事件，而 commit 相当于触发事件
      5. mutation 载荷：在 commit 方法中提交额外的参数
      6. 在 commit 方法中，也可以提交一个对象形式的参数，并且这个对象包含一个叫做 type 的属性,它的值为 mutation 中的某个方法名
      7. 有载荷和对象风格这两种提交 commit 的方式
      8. 在 mutation 中使用异步可能会导致程序很难被调试
    3. getters
      1. 可以理解为 vuex 的计算 computed 属性
      2. 默认 state 作为第一个参数，第二个参数可以为其他 getter
      3. 可以通过属性的方式访问（会缓存结果），也可以通过方法访问（不会缓存结果），相同点是可通过 store.getters 对象进行访问
    4. actions
      1. 相当于能处理异步操作的 mutation
      2. actions 提交的是 mutation
      3. 类似于 vue 实例的 methods 属性，可以书写自己的业务逻辑，或者处理异步方法
      4. Action 函数接收一个与 store 实例具有相同方法和属性的 context 对象参数
      5. 为了简化代码 context 对象可通过解构赋值为{commit}方法来提交 mutation，相当于 const {commit}=context
      6. store.dispatch()方法可以触发 action，它的优点是可以破除异步限制，一般在根组件发出请求
      7. 有载荷和对象风格这两种分发方式
      8. 只有当所有触发函数完成后，返回的 Promise 才会执行
    5. modules
      1. 将 store 分割成模块，每个模块都有自己的 state,mutations,getters,actions 属性
      2. 解决可能的 store 对象臃肿问题
      3. 对于模块内部的 mutation 及 getter 函数，第一个参数为模块的局部状态对象
      4. 对于模块内部的 action，局部状态通过 context.state 暴露出来，而根节点为 context.rootState
      5. 可使用 store.registerModule()方法动态注册模块
      6. 可使用 store.hashModule(modulename)检查模块是否被注册

  7. 可以通过 vue 实例的$store 属性对象进行访问

  8. 使用映射函数的好处是可以减少代码重复书写良，和扩展运算符结合使用

  9. mapState 函数可以帮忙生成 computed 属性，但是它返回的是一个对象，所以还需用对象展开运算符去除里面的所有对象。
