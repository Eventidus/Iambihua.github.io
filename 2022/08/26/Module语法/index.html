Module 语法

  1. ES6 模块可在编译时(即静态阶段)确定模块的编译关系，而 CommonJS 只能在运行时确定模块的依赖关系，所以 ES6 模块可实现静态优化。
  2. ES6 模块支持按需加载需要的模块方法，但 CommonJS 只能加载所有的模块方法。
  3. ES6 模块自动采用严格模式。
  4. export 命令规定模块的对外接口，import 命令用于从模块中导入需要的功能。
  5. export 命令可以输出变量，函数和类。
  6. 通过 import 命令导入的模块，可在模块导入语句的头部提前执行。
  7. 按需导入的变量名应与按需导出的名字一致，
  8. 如果有多个导入相同模块的语句，那么只会执行一次。
  9. 模块的整体加载即 import * as x from ‘xxx’，将模块全部加载进对象 x 中。
  10. export default 命令与 export 命令的区别
    1. 前者的 import 不需要大括号，而后者需要。
    2. 前者在模块中只能使用一次，而后者可以使用多次。
    3. 前者的 import 可以直接起别名，而后者只能使用 as 关键词。
    4. 前者可以同时默认导入和按需导入。
  11. export default 的本周是将后面的值赋给一个叫 default 的对象。
  12. export { x,y } from ‘my_module’相当于如下两行语句
    1. import {x,y} from ‘my_module’;
    2. export {x,y};
  13. 支持 const 声明的常量跨模块共享。
  14. import()函数用来动态加载模块，并且返回一个 Promise 对象。
