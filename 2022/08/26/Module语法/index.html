Module语法

  1. ES6模块可在编译时(即静态阶段)确定模块的编译关系，而CommonJS只能在运行时确定模块的依赖关系，所以ES6模块可实现静态优化。
  2. ES6模块支持按需加载需要的模块方法，但CommonJS只能加载所有的模块方法。
  3. ES6模块自动采用严格模式。
  4. export命令规定模块的对外接口，import命令用于从模块中导入需要的功能。
  5. export命令可以输出变量，函数和类。
  6. 通过import命令导入的模块，可在模块导入语句的头部提前执行。
  7. 按需导入的变量名应与按需导出的名字一致，
  8. 如果有多个导入相同模块的语句，那么只会执行一次。
  9. 模块的整体加载即import * as x from ‘xxx’，将模块全部加载进对象x中。
  10. export default命令与export命令的区别
    1. 前者的import不需要大括号，而后者需要。
    2. 前者在模块中只能使用一次，而后者可以使用多次。
    3. 前者的import可以直接起别名，而后者只能使用as关键词。
    4. 前者可以同时默认导入和按需导入。
  11. export default的本周是将后面的值赋给一个叫default的对象。
  12. export { x,y } from ‘my_module’相当于如下两行语句
    1. import {x,y} from ‘my_module’;
    2. export {x,y};
  13. 支持const声明的常量跨模块共享。
  14. import()函数用来动态加载模块，并且返回一个 Promise 对象。
