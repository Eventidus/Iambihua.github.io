Git进阶学习

本文是Git的进阶学习篇，来源于Git官方网站.
入门篇可点击Git入门学习篇阅读.
进阶学习之起步篇可点击起步.
进阶学习之分支篇可点击分支.

基础

  1. 获取Git仓库
    • git init 本地初始化一个仓库
    • git clone <url> 从远程克隆一个git仓库到本地
    • git clone <url> <dir-name>从远程克隆一个git仓库到本地并指定仓库名
    • git clone -o <originname>从远程克隆的时候同时指定远程仓库的名称
  2. Git文件
    • 查看文件状态
      • git status
      • git status -s以简约的形式输出状态
        • M (modified)表示修改
        • A (added)表示新添加到暂存区的文件
        • ?? 表示未跟踪的文件
    • git忽略文件
      • cat .gitignore 查看.gitignore文件
      • vim .gitignore编写.gitignore文件
      • .gitignore文件编写规范
        • # 开头的内容会被系统忽略
        • 使用glob匹配模式编写文件
        • 以 / 开头表示只忽略当前目录下的文件,非递归删除
        • 目录以 / 结尾，如build/表示忽略所有名为build的文件夹
        • ! 表示已忽略某一类型的文件但不忽略同一类型的某一文件
    • 查看文件差异

      • git diff 查看工作区和暂存区文件的差异

      • git diff --staged 查看暂存区尚未提交的文件与最后一次提交文件的差异

        • git diff --cached 同上

      • 跟踪文件&添加进暂存区

        • git add

      • 提交更新

        • git commit
        • git commit -m " " 提交时同时指定提交说明
        • git commit -a -m" " 非add直接提交所有工作区的文件并指定提交说明

      • 移除文件

        • git rm移除暂存区的文件同时也从工作目录删除
        • git rm -f强制移除暂存区的文件，针对文件已经修改或者已经添加进暂存区的情况
        • git rm --cache仅仅删除暂存区的文件，而不会删除工作目录的文件

      • 重命名文件

        • git mv <oldfile> <newfile>

      • 查看提交历史

        • git log查看提交历史
        • git log -p或git log --patch查看提交历史的同时显示文件的变化内容
        • git log --stat 查看提交历史并且显示被修改的文件数量，被修改的文件以及增加/删除的行数
        • git log  --pretty=oneline只输出哈希值和提交说明

      • 修改提交信息

        • git commit --amend重新提交，以修改上一次的提交说明

      • 撤销暂存区文件的修改

        • git reset HEAD <filename>

      • 撤销工作区文件的修改

        • git checkout --<filename>
  3. Git远程仓库
    • 查看远程仓库
      • git remote输出当前目录所有远程名
      • git remote -v输出远程仓库名以及对应的url地址
      • git remote show <shortname>显示更详细的远程仓库信息
    • 添加远程仓库
      • git remote add <shortname> <url>
    • 拉取远程仓库数据(非合并)
      • git fetch <shortname>shortname是远程仓库简称，可以代替远程url地址
    • 拉取远程仓库数据(合并)
      • git pull
    • 推送到远程仓库
      • git push <origin> <branch>推送到远程仓库的某一分支
      • git push <origin> A:B将本地的A分支推送到远程的B分支
    • 重命名远程仓库
      • git remote rename
    • 移除远程仓库
      • git remote remove
  4. Git标签
    • 列出标签
      • git tag列出所有标签
      • git tag -l或git tag --list同上
    • 创建附注标签
      • git tag -a <tag> -m " "创建附注标签，其中-m表示输入关于附注标签的信息
    • 输出标签信息
      • git tag show <tag>输出关于某标签的信息
    • 创建轻量标签
      • git tag <tag>
    • 后期打标签
      • git tag -a <tag> <commitID>后期打标签，给某次提交打标签
    • 推送标签到远程仓库
      • git push <origin> <tag>推送标签到远程仓库
      • git push <origin> --tags将所有标签推送到远程分支
    • 删除标签
      • git tag -d <tag>删除本地分支的标签
      • git push <origin> :refs/tags/<tag>删除远程仓库的标签
      • git push <origin> --delete <tag>同上
  5. Git别名
    • git config --global alias.br branch给branch别名为br
