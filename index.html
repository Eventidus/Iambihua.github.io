异步操作

  1. JavaScript是单线程模型，JavaScript脚本只在一个线程上运行，这个线程叫主线程。
  2. Web Worker标准，允许JavaScript脚本创建多个线程，但是子线程只受主线程影响，且不得操作DOM。
  3. 同步任务
    1. 在主线程上排队，只有上一个任务执行完成，才会执行下一个任务。
  4. 异步任务
    1. 在任务队列排队，不用管当前任务是否执行完毕，可以直接执行下一个任务。
  5. 任务队列：存放需要执行的所有异步任务。
  6. 异步操作的流程控制
    1. 串行执行，耗时
    2. 并行执行，省时但耗资源
    3. 串行并行结合，省时同时也比较节约资源。

this关键字

  1. this总是返回一个对象。
  2. this的值为属性或方法当前所在的对象(环境)。
  3. 使用场合
    1. 在全局环境下，this的值为顶层环境window。
    2. 在构造函数中，this指向实例对象。
    3. 在对象的方法中，this指向当前方法所在的对象，但如果对一个对象进行赋值时，this的指向就会改变了。
  4. 多层this的情况下，可以考虑使用一个变量指向外层this。
  5. 回调函数中的this指向当前环境中的全局对象。
  6. Function.prototype.call()
    1. 用来指定this的指向。
    2. call方法的参数如果为空,null,或者underfined，则默认指向window对象。
    3. 如果只有一个参数，并且参数为对象，则this指向这个参数对象。
    4. 如果只有一个参数，并且参数为原始值，则this指向这个原始值的包装对象。
    5. 如果有多个参数，则第一个参数为this指向的对象，后面的参数为函数调用时需要的参数。
  7. Function.prototype.apply()
    1. 跟call方法类似，也是用来指定this的指向。
    2. 如果只有一个参数，并且参数为对象，则this指向这个参数对象。
    3. 如果有多个参数，则第一个参数为this要指向的对象，第二个参数为参数数组。
  8. Function.prototype.bind()
    1. bind方法用于将函数中的this绑定到某个对象，并且返回一个新函数。
    2. bind方法也可以用于绑定参数。
    3. 如果只有一个参数，并且参数为对象，则this指向这个参数对象。
    4. 如果有多个参数，则第一个参数为this指向的对象，后面的参数为函数调用时需要的参数。
    5. 注意事项：
      1. 每次调用会返回一个全新的函数

Promise对象

  1. Promise是一种异步编程的解决方案，简单来说就像一种容器，里面保存着某个只有未来才会结束的事件的结果。
  2. Promise的三种状态，pending(进行中),fulfilled(已完成),rejected(已失败)。
  3. Promise的最终状态由异步操作之后的结果决定，并且状态改变之后不会再变。
  4. Promise 内部的错误不会影响到 Promise 外部的代码
  5. Promise的两种状态变更
    1. 由pending到fulfilled。
    2. 由pending到rejected。
  6. 创建Promise实例
    1. 通过构造函数new Promise()来创建一个Promise实例。
    2. 这个构造函数接受一个函数形式的参数，并且这参数接受两个参数resolve和reject。
    3. resolve方法用于把异步操作成功之后的结果作为参数传递过去。
    4. resolve方法的参数也可以是另一个Promise实例，这时候当前的Promise状态由作为resolve参数的Promise实例状态决定。
    5. reject方法用于把异步操作失败之后的报错作为参数传递过去。
  7. then方法
    1. then方法在当前脚本的所有同步任务完成之后才会执行。
    2. 一般用于指示下一步流程
    3. then方法接受两个参数，并且都是可选的，第一个表示状态为resolve时调用，第二个表示状态为reject时调用。
    4. 如果只有一个参数时，就表示状态为resolve时调用。
    5. then方法返回一个promise实例，意味着可以进行链式调用。
  8. catch方法
    1. 它其实相当于只有一个参数，并且状态为reject时的then方法。
    2. 主要是用来捕获错误的。
    3. catch方法返回的也是一个Promise实例。
  9. finally方法
    1. finally方法表示不管状态如何，最后都会执行的方法。
  10. all方法
    1. 将多个Promise实例组合成一个新的Promise实例。
    2. 适用于并行执行多个Promise实例。
    3. 参数为一个数组，并且数组元素都是Promise实例。
    4. 如果数组元素不是Promise实例，会调用resolve()方法转换成Promise实例。
    5. 整合的Promise实例的状态由组合成它的多个Promise实例状态决定
      1. 当所有Promise实例状态都为fulfilled时，这个整合的Promise实例状态为fulfilled，它的结果为一个新数组，并且数组元素为每个promise实例返回的结果。
      2. 当任意一个Promise实例状态为reject时，这个整合的Promise实例状态为fulfilled，会返回碰到的第一个reject实例的返回值。
  11. race方法
    1. 将多个Promise实例组合成一个新的Promise实例P。
    2. 如果数组中的任意一个实例状态率先改变状态，就会将首先改变的实例的返回值传递给实例P的回调函数。
  12. allSettled方法
    1. 当一组异步操作完成后，不管每一个操作是成功还是失败，都进行下一步操作。
    2. 它的参数为原先的数组，数组的元素都是Promise对象。
    3. 它返回一个新的Promise实例对象，有status和value属性。
    4. 只有等到参数数组里的Promise对象都发生状态变更，这个新的Promise实例对象才会发生状态变更，并且状态只能为fulfilled。
  13. any方法
    1. 将多个Promise实例组合成一个新的Promise实例。
    2. 只要有一个Promise实例状态为fulfilled状态，那么这个新的Promise实例的状态为fulfilled
    3. 只有每一个Promise实例都为rejected状态，这个新的Promise实例的状态才为rejected
  14. resolve
    1. 用来将一个对象转换为Promise对象
    2. 参数的四种情况
      1. 如果参数是Promise实例，则原封不动地不做任何修改
      2. 参数是thenable对象(指有then方法的对象)，则将thenable对象转为Promise对象后，再立马执行thenanle对象中的then方法
      3. 如果参数是原始值，则返回一个状态为resolve的Promise实例
      4. 如果不带任何参数，也是直接返回一个resolve状态的Promise实例，需要注意的是这个Promise对象要在本轮循环结束后才会执行
  15. reject()
    1. 返回一个状态为rejected的Promise实例。

package.json介绍

  1. package.json作用
    1. 包管理配置文件，管理项目中所依赖的包的配置
  2. package-lock作用
    1. 本质是锁定(lock)包的版本
  3. 常见属性
    1. script
      1. 定义脚本命令
      2. 每次运行npm run xxx命令时，会创建一个shell脚本，这个脚本会把当前项目的node-modules/.bin子目录写入到系统的Path环境变量中
    2. bin
      1. 将一个包的可执行文件加载到全局环境中
      2. 如果要在全局环境中直接使用，前提是包要安装在全局环境下
    3. type
      1. 用来说明包使用的模块化规范
      2. 如果采用ESM模块规范，则type的值为module
      3. 如果采用CommonJS模块规范，则type的值为commonjs
      4. 如果不想修改type，可以修改文件后缀名
        1. ESModel模块规范后缀名为.mjs
        2. CommonJs模块规范后缀名为.cjs
    4. main&module&browser
      1. 都是用来指定入口文件
      2. main：定义了包的入口文件,支持node和browser环境
      3. module定义了包在ESM规范下的入口文件，支持node和browdeer环境
      4. browser定义了browser环境下的入口文件
      5. 这三个入口文件的加载顺序，一般是ESM规范的文件先行
        1. browser+mjs>module>browser+cjs>main
    5. exports
      1. 定义npm包的所有导出
      2. 条件引用
        1. 引用方式以及模块化文件类型会影响入口文件的引入
        2. require引入.cjs文件
        3. import引入.mjs文件
    6. dependencies&devDependencies
      1. dependencies:项目所需要的依赖
      2. devDependencies:项目所需要的模块，用来提高开发效率
    7. engines
      1. 指定项目能够运行的node以及npm版本
  4. npm扁平化
    1. npm扁平化的原因是npm2过多重复依赖导致的磁盘浪费问题以及window系统下路径过长问题
    2. 缺点
      1. 幽灵依赖问题（引入没有在package.json中声明的依赖）
      2. 存在一个包的多个版本下，只能实现一个版本的扁平化

Module语法

  1. ES6模块可在编译时(即静态阶段)确定模块的编译关系，而CommonJS只能在运行时确定模块的依赖关系，所以ES6模块可实现静态优化。
  2. ES6模块支持按需加载需要的模块方法，但CommonJS只能加载所有的模块方法。
  3. ES6模块自动采用严格模式。
  4. export命令规定模块的对外接口，import命令用于从模块中导入需要的功能。
  5. export命令可以输出变量，函数和类。
  6. 通过import命令导入的模块，可在模块导入语句的头部提前执行。
  7. 按需导入的变量名应与按需导出的名字一致，
  8. 如果有多个导入相同模块的语句，那么只会执行一次。
  9. 模块的整体加载即import * as x from ‘xxx’，将模块全部加载进对象x中。
  10. export default命令与export命令的区别
    1. 前者的import不需要大括号，而后者需要。
    2. 前者在模块中只能使用一次，而后者可以使用多次。
    3. 前者的import可以直接起别名，而后者只能使用as关键词。
    4. 前者可以同时默认导入和按需导入。
  11. export default的本周是将后面的值赋给一个叫default的对象。
  12. export { x,y } from ‘my_module’相当于如下两行语句
    1. import {x,y} from ‘my_module’;
    2. export {x,y};
  13. 支持const声明的常量跨模块共享。
  14. import()函数用来动态加载模块，并且返回一个 Promise 对象。

Module的加载实现

  1. 在浏览器中加载ES6模块
    1. <javascript>标签的两种属性async与defer的相同点是都是异步加载。
    2. <javascript>标签的两种属性async与defer区别
      1. defer：页面渲染完后执行。
      2. async：脚本下载完成后执行，同时中断当前的渲染。
      3. 多个defer脚本按顺序执行，多个async脚本执行顺序不能保证。
    3. 模块顶层的this指向undefined。
    4. 在浏览器中加载ES6模块时，需要在<javascript>标签中指定type=”module”属性。
    5. 浏览器中的ES6模块，默认是defer属性的异步加载。
  2. CommonJs模块和ES6模块的区别
    1. CommonJs模块输出的值的拷贝，ES6模块输出的是值的引用。
    2. CommonJs是运行时加载，而ES6模块是编译输出接口。
    3. commonJS是同步加载，而ES6模块是异步加载。
  3. 在Node中加载ES6模块
    1. 在package.json中在type改为’module’。
    2. 或者将文件名的后缀改为.mjs。
    3. 如果type已经改为’module’,但又想用require命令加载模块，可将当前文件后缀改为.cjs。
    4. ES6模块可以import命令加载CommonJs模块，但CommonJs不能加载ES6模块。
  4. CommonJs模块加载的原理是加载时执行，并且会缓存下来，以后后面再加载时不会再执行。

Generator函数语法

  1. Generator相当于一个状态机，里面封装了多种内部状态
  2. 创建Generator函数时，Function关键字跟函数名之间要有*星号
  3. 调用Generator函数后，不会执行里面的代码，而是生成一个迭代器对象
  4. 返回的迭代器对象可以反复使用next()方法迭代内部状态
  5. 使用next方法会返回一个对象，这个对象有两个属性
    1. value：当前状态的值，即yield表达式的值
    2. done：迭代是否结束
  6. 每次遇到yield语句并执行完成，函数会暂停执行
  7. 函数可以有多个yield语句，但只能有一个yield语句
  8. next方法带参数时，代表上一个yield返回值的结果为该参数的值
  9. 可以用for-of循环迭代Generator函数返回的迭代器对象，如果对象done属性为true，不会输出该对象
  10. yield表达式如果用在另一个表达式中，必须放在圆括号里
  11. Generator.prototype.throw()
    1. throw方法是用来在函数外部抛出错误
    2. 注意只能用throw方法抛出，而不能用throw命令关键字抛出，否则Generator函数内部不会捕获错误
  12. Generator.prototype.return()
    1. return语句可以返回一个值，并且终止此后的迭代操作
    2. yield参数为要返回的值，如果为空，则返回underfined
    3. 如果Generator函数内有try-finally语句块，则会直接执行finally语句内的值，再返回return指定的返回值
  13. yield*表达式
    1. 用来在一个Generator函数里执行另一个Generator函数
    2. 任何实现Iterator接口的数据结构也可以用yield表达式迭代

ECMAScript6

  1. JavaScript为动态类型语言。
  2. 静态语言：在编译阶段便检查数据类型。动态语言：在执行期间才检查语言的数据类型。
  3. 扩展运算符
    1. 取出参数对象的所有可遍历属性，拷贝到当前对象中。
  4. 常用方法
    1. filter()
      1. 语法:array.filter(function(currentValue,index,arr), thisValue)
      2. 功能：返回一个过滤符合要求的元素并且组成新数组
    2. map()
      1. 功能：返回回调函数对每个原数组元素进行操作(比如加1)之后组成的新数组，原数组不变。
    3. foreach()
      1. 遍历数组并且对中的每个元素进行操作。
    4. shift()
      1. 删除数组的第一个元素，并且返回该元素的值。
      2. 可改变数组的长度。
    5. splice()
      1. 删除或增加或替换元素
      2. 第一个参数为修改开始的位置
      3. 第二个参数为删除元素个数
      4. 第三个参数为要增加或者替换的元素
    6. split()
      1. 将字符串分割成字符串数组，字符串的每一个字母都可以是一个数组元素
      2. 参数是一种分隔符
    7. join()
      1. 将数组组合成一个字符串
      2. 参数是一种分隔符，让生成的字符串根据这个分隔符分隔
    8. indexOf()
      1. 数组
        1. 返回某个在数组中的索引，如果没有找到则返回-1
      2. 字符串
        1. 返回第一个子串在字符串出现的索引，如果没有则返回-1
  5. 计算属性概念
    1. 一个对象的属性名从一个变量中获取，并且这个变量用[]表示。

async函数

  1. async函数的执行原理就是将Generator函数与自动执行器包装在一个函数里
  2. async相当于Generator的语法糖
  3. async自带执行器
  4. async返回一个Promise对象
  5. async函数调用then方法时，如果asyn函数有return返回值，那么这个返回值会成为then方法的回调函数的参数
  6. async函数内部抛出的错误，会使返回的Promise对象变为rejected状态
  7. await命令后面通常是promise对象
  8. async函数返回的Promise对象只有等所有await命令执行完成之后才会发生状态变更
  9. 只要有一个await命令的Promise对象变为reject状态，后面的await命令 不会执行
  10. 给当前await命令语句写进try-catch结构，或者调用catch()方法，这样如果当前await命令出现错误，不会影响后面的await命令继续执行
