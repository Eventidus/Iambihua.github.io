Java 核心类

1.String

  • String 字符串内部是通过一个char[]表示的。
  • String 字符串不可变性的实现：通过内部的private final char[]实现。
  • 字符串的比较
    • 用equals()方法来比较，而不用==。
    • Java 编译器在编译期间，会把拥有相同字符串的对象视为同一个对象并且放入常量池，所以用==比较字符串得出的结果会有巧合性。
    • 忽略大小写比较字符串用equalsIgnoreCase()方法。
  • 判断是否包含子串用contains()方法，参数为CharSequence类，为 String 的子类。
  • 搜索字串
    • indexOf()方法返回字串第一次出现在前半部分的索引。
    • lastIndexOf()方法返回子串第一次出现在后半部分的索引。

Java注解

1.注解

  • 注解(Annotation)是一种特殊的注释,作用于Java源码的类，方法，字段，参数前。
  • 注解与代码的关系：从JVM的角度看，注解本身对代码的逻辑没有任何影响。
  • 注解分类
    • 作用于编译器，但不会进入.class件，编译后不复存在,例如
      • @override告诉编译器检查是否进行了正确的代码覆写
      • @suppressWarning告诉编译器忽略当前产生的编译警告。
    • 作用于class文件，但加载后不会存在于内存中。
    • 作用于JVM，在程序运行时能够读取，例如
      • 配置了@PostConstruct参数的方法在调用构造方法时被自动调用。
  • 注解的配置参数必须是常量，注解的配置参数可以有默认值。

《习惯的力量》

05/23
见字如晤
书名: 习惯的力量
作者: [美]查尔斯·杜希格
出版年份: 2013-3
评分: 8.0

前些天读了查尔斯·都希格的《习惯的力量》一书，译者是吴奕俊,曹烨。

我并没有读完这本书，相反只是阅读了个人的习惯这部分，其余两部分是成功的组织机构的习惯和社会的习惯。

我觉得后两部分暂时用不到，自己目前也没有那么多时间阅读，所以先放着，以后如果有需要，还是会拿出来再看看的。

这本书是偏理论的书，但读来并不是那么深奥难懂，而且插入了大量真实例子来说明原理。

读完后我知道了习惯是如何产生的，以及如何改变习惯。

但我觉得读完后理解依然不够深刻，所以我打算写一篇关于这本书的个人的习惯部分的读书笔记。

一方面通过写作加深理解，另一方面也好做个记录供以后大家和自己阅读。

Markdown 介绍

什么是 Markdown?它是一种轻量级的文本标记语言，使用它可以更方便地编辑文档。

很多网站都利用它来撰写文档或用于在论坛发消息等等。

说到这大家可能就像在听天书一样，因为搞不懂专业用语。

事实上这并不是什么计算机编程语言，而是通用的普通人都可以学的标记语言，贱简单呐。
