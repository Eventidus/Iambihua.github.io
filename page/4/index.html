在我的认知观里，一款优秀的软件应该满足以下特性，另外我也整理了以下几个我觉得生活中很实用的软件。

  • 一个软件就应该做好一件事。
  • 不要太多臃肿的功能。
  • 简洁高效，不要太花里胡哨。

1.Office

微软WORD,EXCEL,PPT三合一版Office，简洁无广告，它目前主要提供以下功能，并且全部免费！自从下载Android版Office后，我就把WPS卸载了，其一是广告太多，看着就烦;其二是有很多功能需要开通会员。这个Office虽然功能不多，但是应付平时的需要应该还算够用了。

  • 在手机和计算机之间共享文件，只需扫码即可把文件传送到计算机。
  • 图像到文本，从图像提取文本。
  • 图片到表，将表从图片提取到Excel中。
  • 文档到PDF,将文档转换为PDF。
  • 图片到PDF，基于图片创建PDF。

2.Google Translate

谷歌公司开发，也是一款很强大很简洁的翻译工具，主要的是还支持离线翻译，没有任何广告，是真正的符合了我在上面总结的全部特型。

  • 拍照或相册翻译。
  • 手写翻译。
  • 对话和语音翻译。

3.Snapseed

也是谷歌公司开发，自带滤镜功能，并且滤镜类型非常多，最重要的还是全部免费，个人使用的体验是，上手非常简单，纯小白操作，小白也能P出高质量图片，这就满足了简洁高效的特性。

  • 样式丰富。
  • 工具丰富，排列整齐。

4.酷安

这是一款专门安装应用程序app的软件，相当于手机系统自带的软件商店，有时候应用商店无法找到的软件可以在这里安装，基本都能找到。不只提供软件安装功能，它还有很多功能模块。个人使用体验是：软件总体来说比较臃肿，但还是能够接受，功能繁多，眼花缭乱的感觉，平时主要用途还是安装软件吧。

  • 话题丰富，基本可以找到自己感兴趣的话题。
  • 多类型图片，图片质量高。
  • 数码模块，以后买数码产品可以看上面评论参考一下。
  • 评论丰富，有一定的可信度。
  • 丰富的软件生态。

《习惯的力量》

2021-06-05
书籍

前些天读了查尔斯·都希格的《习惯的力量》一书，译者是吴奕俊,曹烨。

我并没有读完这本书，相反只是阅读了个人的习惯这部分，其余两部分是成功的组织机构的习惯和社会的习惯。

我觉得后两部分暂时用不到，自己目前也没有那么多时间阅读，所以先放着，以后如果有需要，还是会拿出来再看看的。

这本书是偏理论的书，但读来并不是那么深奥难懂，而且插入了大量真实例子来说明原理。

读完后我知道了习惯是如何产生的，以及如何改变习惯。

但我觉得读完后理解依然不够深刻，所以我打算写一篇关于这本书的个人的习惯部分的读书笔记。

一方面通过写作加深理解，另一方面也好做个记录供以后大家和自己阅读。

习惯回路

习惯是如何产生的？在我们人脑中，有一个至关重要的区域——基底核。它与人的习惯紧密相关。

如果某人的大脑除了基底核外关于记忆的区域受损，那么这个人肯定会忘记很多事。

比如他会不记得回家的路，或者不知道5分钟前做过什么等等。

然而他却记得回家的路，只要环境一直没受影响，这到底是为什么？

这就是习惯的力量，它的力量很强大，这就是为什么记忆丧失的人依然知道回家的路的原因。

但它的力量同时又很脆弱，因为一旦环境改变，记忆丧失之人就不能辨认周围环境的暗示，再也找不到回家的路。

我想很少有人注意到习惯这个特性，以至于以为它是生活中很平常的，日积月累的结果，并不会关心它实际是怎样产生的。

关于习惯的产生，我们首先需要理解习惯回路。

习惯回路：暗示—>惯性行为—>奖赏。

按上面的解释来看，习惯回路还是比较容易理解的，无非就是三部分，三者缺一不可，否则很难形成长期的习惯。

举个例子：一个人受到暗示或刺激，比如一个人听到闹钟响，他知道他该起床刷牙了，于是他的惯常行为是他去刷牙，最后他的奖赏结果是他体会到了刷牙时那种有薄荷味刺激性的感觉。

然而光靠习惯回路还远远不够的，因为在你做习惯性行为前，你还得渴求着什么。

渴求

个人的欲望或者渴求对习惯回路有什么作用？很明显它是起驱动作用的。

因为当你在做某个习惯性行为前，你肯定是在心里渴求做完这种事的奖赏，比如体会到冰淇淋的冰凉快感，亦或者是体会到刷牙时舌头的刺激。

否则你很难心甘情愿地去干这件事，因为你并没有特别想要完成某事后得到奖赏的欲望，进而就很难形成长期的习惯行为。

黄金法则

黄金法则：用同样的暗示，同样的奖赏，改变习惯回路中间的惯常行为。

如果你有某种坏习惯，你知道它的危害，但很难改变它，比如酗酒，暴饮暴食等，不是劝止喝酒那么简单。

然而如果你试着利用黄金法则去改变旧习惯，我想是最有效的。

因为它提供了同样的暗示，也提供了同样的奖赏，我们只需要改变中间的惯常行为，几乎是最轻松最容易的改变方式。

旧习惯很难根除，因为我们对某种行为的奖赏结果依然很渴望，所以只要这种渴望依然存在，我们就难以劝止自己不做这种习惯性行为。

这时候比较好的办法就是不遏制忽略这种渴求，相反用其他合适良好的能够提供同样奖赏的行为去替代之前的行为。

信仰

但有时候我们却不太愿意相信自己能改变，对这种改变心存犹豫，这样势必会造成形成持续性习惯行为的中断。

所以习惯的成功改变，还需要一定的心理基础，那就是信仰，相信自己能改变的信仰。

对自己增强信任的方式有多种，每一种可能都会奏效，但我觉得，集体给予自己的信任，会更长久更坚韧。

当一个人给予自己鼓励时，很难树立起长久强大的信任。

但是如果一个集体给予自己的信任，也许比任何方式来得更强大，更历久弥新。

也许这就是集体力量的伟大之处，有超强的凝聚力的同时，给予每个成员莫大的信任。

总结：了解习惯产生的原因即习惯回路，以渴求驱动行为，通过习惯改变的黄金法则，加上强大的信仰，任何人都可以改变旧习惯。

全文完。

Coursera课程

Learning how to learn -prat 1

一.两种思考模式

1.专注模式

  • 当我们大脑在专心思考某个问题时大脑所处的模式，是比较紧密的思考模式。

2.发散模式

  • 当我们大脑在多广度思考时大脑所处的模式，是一种概括性的思考模式。

3.思考模式切换

  • 在大脑中的某一时刻，专注模式和发散模式只能同时存在一种模式。

  • 学习时，最好的利用大脑思考模式的方法是两种模式相互切换。

  • 有时候发散模式有助于梳理知识，增强对相关知识的理解性。

4.类比和比喻

  • 类比和比喻两种有效的理解知识的方式，如果自己试着用类比或比喻的方式去解释知识，或者对概念知识进行比较会更能理解知识。

二.番茄工作法

番茄工作法

  • 番茄工作法是一种有效的应对拖延症的方法。

  • 当我们要逃避做某件事时，比如做作业或者践行学习计划，我们可以设定25分钟的学习时间，并且排除其他干扰，最后休息十分钟，必要的话，可以在完成后给自己一些小奖励。

三.神经元之间的连接与学习的关系

联系

  • 当我们学习新概念特别是抽象性东西时，经常练习是必要的。

  • 因为我们在思考或者练习时，从生物学角度来看，相当于增强大脑神经元之间的连接。

  • 对抽象性概念的东西练习的越多，就会越理解，神经元之间的连接也会更紧密。

四.工作记忆与长期记忆

1.长期记忆：

  • 相当于一个存储仓库，里面存储的记忆是长期的。

2.工作记忆

  • 理解新概念，新事物的时被调动的记忆方式，这方式的记忆是短暂的。

3.长期记忆与工作记忆之间的联系

  • 长期记忆与工作记忆是相互关联的，长期记忆是由工作记忆的一部分转化而来的。

  • 工作记忆要想转化为长期记忆，就得经过多次的复习。

  • 间隔性的复习方式更容易使工作记忆转化为长期记忆。

五.睡眠的至关重要性

1.清理大脑内部的毒素。

  • 睡眠时，脑细胞会收缩，这样会增加脑细胞间的间距，液体会在间距中流动并冲走毒素。

2.帮助记忆。

  • 睡眠时大脑会将学习和思考过的想法以及概念进行整理 ，它会清除掉一些记忆中不太重要的部分 ，同时增强你需要或想要记住的区域的记忆。

  • 在睡前回想学过的东西，更能加深记忆。

  • 平时多思考，这样脑海中会拥有更多的理解，最后借助大脑的睡眠作用加深这些知识的理解与记忆。

六. 对Terrence Sejnowski院士采访内容的总结

总结

  • 实践是最好的学习方式，因为它会调动大脑的积极性。

  • 进入发散模式的方式可以是跑步或者户外锻炼等，最好与锻炼结合起来，因为在进行发散思考的同时锻炼了身体。

  • 在一个富有创造力的环境中学习更容易激发自己的创造力，并且提高学习的积极性和专注力。

1. 什么是组块

组块是大脑中的一种神经信息片段。

  • 组块整体化可以让更多信息联系起来。

  • 更易于记忆

Git进阶学习

一.入门

  1. 版本控制概念
    • 版本控制是一种记录文件内容变动的文件系统。
  2. 三种版本控制系统
    • 本地版本控制系统
      • 缺点是不方便协作工作，并且有丢失更改历史的风险。
    • 集中式版本控制系统
      • 缺点是容易出现服务器单点故障，如此会导致本地无法从服务器获取或上传文件。
    • 分布式版本控制系统
  3. Git介绍
    • 每一次更改生成文件快照，这一点不同于其他版本控制系统，其他版本控制系统保存的仅仅是文件的变化内容。
    • 除了上传和更新本地库，其他所有操作几乎是在本地进行，这应该是git操作快速的根本原因。
    • 通过哈希值来索引文件，而不是文件名。
    • 三种状态
      • 工作区。
      • 暂存区(又叫索引)
      • 本地版本库
  4. 环境配置
    • 配置用户信息
      • git config --global user.name abc 配置用户名。
      • git config --global user.email xxx.example.com 配置用户邮箱
      • 在每一次提交上都会写入配置的默认用户名和邮箱。
      • 如若在特定的项目使用不同的用户名和邮箱，可以不带上–global参数。
    • 配置默认编辑器
      • git config - - global core.editor notepad++配置notepad++为默认编辑器。
    • 查看配置信息
      • git config --list
  5. 获取命令信息
    • git help <verb> 获取命令手册(会自动打开浏览器)。
    • git <verb> --help 获取命令手册
    • git <verb> -h 获取命令选项列表

分支

  1. Git仓库对象
    • blob对象保存着文件快照
    • 树对象记录着目录结构和blob对象索引
    • 提交对象保存着树对象索引和提交信息
  2. Git分支
    • 创建分支
      • git branch <branch>
      • git checkout -b <branch>创建分支并切换
    • 查看分支的指向
      • git log --oneline --decorate查看分支指向的提交点
    • 切换分支
      • git checkout  <branch>切换分支
    • 分支合并
      • git merge <branch>
    • 删除分支
      • git branch -d <branch>
    • 查看分支
      • git branch查看当前项目的所有分支
      • git branch -v查看当前项目的所有分支以及它们的前七位哈希值和提交说明
      • git branch --merged查看已经合并到当前分支的分支
      • git branch --no-merged查看未合并到当前分支的分支
  3. Git远程分支
    • 创建跟踪分支
      • 从一个远程分支检出的本地分支叫做跟踪分支，它跟踪的分支叫做上游分支
      • 设置跟踪分支的目的是在这个分支上进行工作。
      • 从远程仓库克隆时会自动创建一个跟踪分支 ，即master分支。
      • git checkout -b <branch> <origin>/<branch>
      • git branch --track <origin>/<branch>
      • git checkout <branch>如果检出的本地分支不存在而刚好存在一个与其名字匹配的远程分支，则在检出时会自动创建跟踪分支
    • 重命名跟踪分支
      • git checkout -b <branch> <origin>/<branch>
    • 修改被跟踪的上游分支
      • git branch -u <origin>/<branch>
    • 查看跟踪分支
      • git branch -vv
    • 删除远程分支
      • git push <origin> --delete <branch>
  4. Git变基
    • 变基是对两个已修改提交的处理，不同于合并，变基是把修改和补丁应用到另一个分支上，而合并是对两个提交快照的内容合并。
    • 变基与合并有截然不同的的提交历史，变基是线性的，而合并是并行的。
    • git rebase <branch>
    • 流程：先变基后合并。
