CORS通信

  1. CORS（Cross-Origin-Resource-Sharing），中文名跨域资源共享。
  2. CORS的作用是允许浏览器向跨域服务器发送XMLHttpRequest（XHR）。
  3. 实现CORS跨域通信，主要看服务器是否实现CORS接口，也就是说主要在服务器端配置
  4. 服务器端配置CORS跨域的方法是设置Access-Control-Allow-Origin
  5. CORS可分为两种请求：
    1. 简答请求
      1. 简答请求需要请求方法和头信息字段都满足条件
      2. 请求方法
        1. HEAD
        2. POST
        3. GET
      3. 头信息字段
        1. Accept
        2. Content-Language
        3. Accept-Language
        4. Last-Event-ID
        5. Content-Type：只限于三个值application/x-www-form-urlencoded、multipart/form-data、text/plain
      4. 不满足上面两个条件的，称为非简单请求。
      5. CORS通信的基本流程：
        1. 浏览器发起CORS请求，并且在头信息增加origin字段，表示要发起跨域请求的域。
        2. 服务器判断浏览器发过来的origin是否在许可范围内。
        3. 如不在许可范围内，则发送正常的HTTP回应，但不会包含Access-Control-Allow-Origin字段。
        4. 如在许可范围内，发送的HTTP回应会包含以下字段
          1. Access-Control-Allow-Origin字段： 表示服务器可允许跨域的域名，如果是*，则表示可接受任意域名。
          2. Access-Control-Allow-Credentials：表示是否允许发送cookie。
          3. Access-Control-Expose-Headers：表示是否返回其他非基本字段。
        5. 要想发送cookie给服务器，开发者需要将xhr.withCredentials设置为true，注意有些浏览器的xhr.withCredentials默认为false。
    2. 非简单请求
      1. 非简单请求在发送CORS请求前，需要先发出预检请求。
      2. 预检请求是浏览器询问是否接受当前网页的域名，以及可接受哪些HTTP方法和字段。
      3. 预检请求的请求方法是OPTIONS。
      4. 预检请求的头信息包含的部分字段如下
        1. origin
        2. Access-Control-Request-Method，发送CORS请求用到的方法。
        3. Access-Control-Requset-Headers，发送CORS请求额外发送的头部信息
      5. 如通过预检请求，则每次的CORS请求就跟简单请求一样。
      6. CORS和JSONP的比较
        1. JSONP只支持Get请求，而CORS支持所有类型的请求。
        2. JSONP可以在老式浏览器上运行，也可以向不支持CORS的网站上请求数据。

VueRouter

  1. 导入vue及vue-router库
  2. 安装vue-router插件
  3. 导入路由组件
  4. 创建路由数组即routes数组，数组一般有
    1. path属性定义访问路径
    2. component属性指定组件名
    3. redirect属性用于路由重定向
    4. meta属性是一个对象，为路由记录添加其他额外信息，通过这个信息以实现控制路由的访问等功能
  5. 创建路由器对象即通过new VueRouter()构造函数,将routes数组传入构造函数参数里,其他参数：mode:指定url地址是否带有 # 号
  6. 导出路由配置文件
  7. 在main.js主入口文件中引入路由规则文件，并且注册路由
  8. vue路由是一种键值对，它表示的是路径与路由组件之间的映射
  9. 路由的$route属性用于获取路由相关信息如query,params，fullPath(可用于组件的隐藏或显示)等
    1. query查询参数，也就是url地址?后面的参数,不需要预先占位。
    2. params路径参数相当于url地址的一部分，用于动态指定url地址，需要预先占位。
  10. 路由传参的时候，如何指定params参数可传也可不传
    1. 在配置路由规则的时候，在路由占位的后面加上一个问号。
    2. 如果同时提供了path和params，那么params会被省略，但是query不会。
    3. 如果同时提供了name和params，那么params可以正常工作。
  11. 路由的$router属性用于编程式导航进行路由跳转(Push|replace)等
  12. 路由有声明式导航(router-link)和编程式导航，其中声明式导航必须有to属性
  13. 三种路由跳转方法
    1. push()方法会在路由历史栈中添加一条新的记录
    2. replace()方法只是替换，而不会向历史栈中添加一条新的记录
    3. go()表示在历史栈中前进或后退多步
  14. 路由的前进或者后退
    1. router.forward()前进一步
    2. router.back()后退一步
  15. 三种使用props传参的方式
    1. 布尔模式：props=true,这时props的值为params
    2. 对象模式：props的值为一个对象
    3. 函数模式：返回一个props对，route => ({ })

Vuex

  1. 作用：集中管理项目中组件公用的数据
  2. 使用场景：项目过大，组件多，并且数据量多的情况下可以使用vuex
  3. vuex的状态是响应式的
  4. mutations不能直接更改状态
  5. Vuex使用步骤：
    1. 创建store目录，在这个目录下创建index.js文件
    2. 导入Vue及Vuex库
    3. 安装Vuex库，即Vue.use(Vuex)
    4. 创建vuex实例，通过new Vuex.Store({})方法
    5. 向外导出Vuex实例
    6. 在入口文件中导入vuex实例
    7. 注册vuex实例
  6. vuex的几个核心概念
    1. state
      1. 类似于vue的data属性，存储数据源
    2. mutations
      1. mutation是同步事务
      2. 类似于vue实例的methods属性，修改数据源状态，但不能处理异步方法
      3. 不能直接修改state的值，可在子组件中通过调用this.$store.commit(‘xxx’)来更改store中state的值
      4. mutation相当于注册事件，而commit相当于触发事件
      5. mutation载荷：在commit方法中提交额外的参数
      6. 在commit方法中，也可以提交一个对象形式的参数，并且这个对象包含一个叫做type的属性,它的值为mutation中的某个方法名
      7. 有载荷和对象风格这两种提交commit的方式
      8. 在mutation中使用异步可能会导致程序很难被调试
    3. getters
      1. 可以理解为vuex的计算computed属性
      2. 默认state作为第一个参数，第二个参数可以为其他getter
      3. 可以通过属性的方式访问（会缓存结果），也可以通过方法访问（不会缓存结果），相同点是可通过store.getters对象进行访问
    4. actions
      1. 相当于能处理异步操作的mutation
      2. actions提交的是mutation
      3. 类似于vue实例的methods属性，可以书写自己的业务逻辑，或者处理异步方法
      4. Action函数接收一个与store实例具有相同方法和属性的context对象参数
      5. 为了简化代码context对象可通过解构赋值为{commit}方法来提交mutation，相当于const {commit}=context
      6. store.dispatch()方法可以触发action，它的优点是可以破除异步限制，一般在根组件发出请求
      7. 有载荷和对象风格这两种分发方式
      8. 只有当所有触发函数完成后，返回的Promise才会执行
    5. modules
      1. 将store分割成模块，每个模块都有自己的state,mutations,getters,actions属性
      2. 解决可能的store对象臃肿问题
      3. 对于模块内部的mutation及getter函数，第一个参数为模块的局部状态对象
      4. 对于模块内部的action，局部状态通过context.state暴露出来，而根节点为context.rootState
      5. 可使用store.registerModule()方法动态注册模块
      6. 可使用store.hashModule(modulename)检查模块是否被注册
  7. 可以通过vue实例的$store属性对象进行访问
  8. 使用映射函数的好处是可以减少代码重复书写良，和扩展运算符结合使用
  9. mapState函数可以帮忙生成computed属性，但是它返回的是一个对象，所以还需用对象展开运算符去除里面的所有对象。

语义化版本2.0.0介绍

简介

语义化版本是为了解决软件开发过程中依赖关系混乱或者依赖锁死的情况。

它的标准格式如下：

x.y.z

其中x表示主版本号，y表示次版本号，z表示修订号。

对这几种版本号的解释如下：

X 主版本号，表示做了不向下兼容的修改时递增。

Y 次版本号，表示做了向下兼容的新增及修改时递增。

Z 修订号，表示做了向下兼容的问题修复时递增。

Java反射

1.Class类

  • 反射：通过Class实例获取相关类信息的方法称为反射。

  • JVM会在执行过程中动态加载类到内存，并为其创建一个Class类型的实例，并且该Class实例只能在JVM内部创建。

  • 如何获取一个Class实例

    • 通过一个类的静态变量class获取，如String.class。
    • 通过实例变量调用getClass()方法。
    • 通过Class类的静态Class.forName()方法，例Class.forName(String.lang.String),其中参数中的类名要为完整类名。

  • Class实例的比较，可以用==比较，原因是Class实例在JVM中是唯一的。

  • 准确判断两个类型是否为同一类型可用==比较。

  • 数组例如String[ ]也可以创建Class实例，而且不同于String.class。

  • 此外，JVM也为基本类型创建了Class实例,例如int.class。

本文是Java的面向对象篇，学习于廖雪峰的Java教程.

面向对象基础

1.方法

  • private方法和private字段不能外部类访问。
  • private方法和private字段是用来方便自身类使用的，为了保证类的封装性。
  • this 变量指当前实例。
  • 局部变量的优先级高于字段，可变参数也是局部变量。
  • 可变参数的定义：类型... 变量名，相当于数组。

2.构造方法

  • 构造方法名与类名相同，构造方法无返回值，必须用new操作符调用。
  • 未自定义构造方法时，系统会默认创建一个无参数构造方法。
  • 如果已经自定义构造方法，系统将不再默认创建构造方法。
  • 引用类型的字段默认是null，数值类型的字段是默认值，int类型默认值是0，布尔类型默认值是false。
  • 字段的最终赋值结果由构造方法中的赋值语句决定。
  • 一个类中可以有多构造方法，只是参数列表不同。

3.重载方法

  • 方法名相同但参数列表不同的方法叫(overload)重载方法，返回类型通常是相同的。
  • 重载方法的目的是实现相似的功能但得出不同的结果。
  • 方法签名: 方法名和参数列表。
