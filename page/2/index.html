async函数

  1. async函数的执行原理就是将Generator函数与自动执行器包装在一个函数里
  2. async相当于Generator的语法糖
  3. async自带执行器
  4. async返回一个Promise对象
  5. async函数调用then方法时，如果asyn函数有return返回值，那么这个返回值会成为then方法的回调函数的参数
  6. async函数内部抛出的错误，会使返回的Promise对象变为rejected状态
  7. await命令后面通常是promise对象
  8. async函数返回的Promise对象只有等所有await命令执行完成之后才会发生状态变更
  9. 只要有一个await命令的Promise对象变为reject状态，后面的await命令 不会执行
  10. 给当前await命令语句写进try-catch结构，或者调用catch()方法，这样如果当前await命令出现错误，不会影响后面的await命令继续执行

Vuex

  1. 作用：集中管理项目中组件公用的数据

  2. 使用场景：项目过大，组件多，并且数据量多的情况下可以使用 vuex

  4. mutations 不能直接更改状态

npm 常见命令

一.npm&yarn 介绍

npm 是一个基于 Node.js 的包管理工具。
yarn 是 Facebook、Google、Exponent 和 Tilde 联合推出了一个新的 包管理工具,与 npm 相比有些优势。
npm 在安装 Node.js 时会被一并安装，yarn 则需要额外安装。

二.npm&cnpm 的关系

1.介绍

由于 npm 服务器在国外，所以在国内安装速度不是很理想，于是淘宝团队开发了一个 npmjs.org 的镜像，可以有效解决下载过于缓慢的问题。这是淘宝团队对它的介绍。

“这是一个完整 npmjs.org 镜像，你可以用此代替官方版本(只读)，同步频率目前为 10 分钟 一次以保证尽量与官方服务同步。”

2.安装

npm install -g cnpm --registry=https://registry.npm.taobao.org

Java反射

1.Class类

  • 反射：通过Class实例获取相关类信息的方法称为反射。

  • JVM会在执行过程中动态加载类到内存，并为其创建一个Class类型的实例，并且该Class实例只能在JVM内部创建。

  • 如何获取一个Class实例

    • 通过一个类的静态变量class获取，如String.class。
    • 通过实例变量调用getClass()方法。
    • 通过Class类的静态Class.forName()方法，例Class.forName(String.lang.String),其中参数中的类名要为完整类名。

  • Class实例的比较，可以用==比较，原因是Class实例在JVM中是唯一的。

  • 准确判断两个类型是否为同一类型可用==比较。

  • 数组例如String[ ]也可以创建Class实例，而且不同于String.class。

  • 此外，JVM也为基本类型创建了Class实例,例如int.class。

一.两种思考模式

1.专注模式

  • 当我们大脑在专心思考某个问题时大脑所处的模式，是比较紧密的思考模式。

2.发散模式

  • 当我们大脑在多广度思考时大脑所处的模式，是一种概括性的思考模式。

3.思考模式切换

  • 在大脑中的某一时刻，专注模式和发散模式只能同时存在一种模式。

  • 学习时，最好的利用大脑思考模式的方法是两种模式相互切换。

  • 有时候发散模式有助于梳理知识，增强对相关知识的理解性。
